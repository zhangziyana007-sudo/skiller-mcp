## 什么是社区功能？

Skiller 的社区功能让你可以：

- **浏览** 其他人分享的 AI 技能
- **一键安装** 到你的 Cursor 中使用
- **上传分享** 你自己的技能给其他人

每个人的技能都存放在 GitHub 仓库中，通过「订阅源」互相发现。

**打个比方**：就像关注别人的公众号一样，你添加别人的技能仓库地址，就能看到他们分享的所有技能。

## 第一步：安装 Skiller

### 1.1 克隆项目

```bash
git clone https://github.com/zhangziyana007-sudo/skiller-mcp.git ~/.cursor/skiller
cd ~/.cursor/skiller
npm install
```

### 1.2 配置 Cursor MCP

打开你的 Cursor 配置文件 `~/.cursor/mcp.json`，添加：

```json
{
  "mcpServers": {
    "skiller": {
      "command": "npx",
      "args": ["tsx", "/你的HOME路径/.cursor/skiller/src/index.ts"]
    }
  }
}
```

> 把 `/你的HOME路径/` 替换成你的实际路径，比如 `/home/username/` 或 `/Users/username/`

### 1.3 重启 Cursor

重启后，在对话中输入「搜索技能」或「列出技能分类」，如果 Skiller 正常响应，说明安装成功！

## 第二步：打开 Dashboard

启动可视化面板：

```bash
cd ~/.cursor/skiller
npm run dashboard
```

然后在浏览器中打开：**http://localhost:3737**

你会看到一个蜡笔小新风格的技能管理界面 🎨

## 第三步：浏览和安装社区技能

### 3.1 进入社区页面

点击顶部导航栏的 **🌍 社区** 按钮。

**首次使用？** 你会看到设置页面，需要先配置。

### 3.2 配置社区（首次）

如果你只是想**浏览和安装**别人的技能（不上传），操作非常简单：

| 字段 | 填写内容 | 是否必填 |
|------|---------|---------|
| 社区仓库 | 留空 | 否 |
| GitHub Token | 留空 | 否 |
| 作者名称 | 你的昵称 | 否 |

点击「保存配置」后，直接跳到 **[第四步：订阅其他人的技能库](#第四步订阅其他人的技能库)**。

> 只想看别人的技能？不需要配置任何东西，直接添加订阅源就行！

## 第四步：订阅其他人的技能库

这是最常用的功能 —— 添加别人的技能仓库，就能看到并安装他们的技能。

### 4.1 进入订阅外部GitHub仓库

左侧导航 → 点击 **📡 订阅外部GitHub仓库**。

### 4.2 添加订阅源

在「添加新订阅源」区域填写：

| 字段 | 示例 | 说明 |
|------|------|------|
| 仓库地址 | `zhangziyana007-sudo/skiller-community` | 别人告诉你的仓库地址 |
| 显示名称 | `zizaya 的技能库` | 自定义名字，方便区分 |
| 分支 | `main` | 一般默认即可 |
| 技能目录 | `skills` | 一般默认即可 |

点击 **➕ 添加订阅源**。

### 4.3 浏览技能

回到 **🌐 浏览技能** 标签，就能看到这个仓库中的所有技能了！

每个技能卡片上有：
- **⬇️ 安装** —— 一键安装到你的 Cursor
- **👁️ 预览** —— 查看 SKILL.md 的完整内容
- **🔗 GitHub** —— 打开 GitHub 页面

### 4.4 安装技能

点击 **⬇️ 安装** 按钮，技能会自动下载到 `~/.cursor/skills/` 目录。

安装成功后，按钮变成 **✓ 已安装**。在 Cursor 中对话时就可以使用了！

## 第五步：分享你的技能

想把自己的技能分享给别人？需要额外配置。

### 5.1 创建你自己的 GitHub 仓库

1. 打开 https://github.com/new
2. 仓库名填写：`skill-community`（或你喜欢的名字）
3. 选择 **Public**（公开，这样别人才能看到）
4. 勾选 **Add a README file**
5. 点击 **Create repository**

创建后，在仓库中手动创建一个 `skills/` 文件夹（可以在 GitHub 网页上点 "Add file" → "Create new file" → 输入 `skills/.gitkeep`）。

### 5.2 创建 GitHub Token

1. 打开 https://github.com/settings/tokens/new
2. Note 填 `skiller`
3. 勾选 `repo` 权限
4. 点击 **Generate token**
5. **复制 Token**（只显示一次！）

### 5.3 在 Dashboard 中配置

社区页面 → **⚙️ 设置** 标签：

| 字段 | 填写 |
|------|------|
| 社区仓库 | `你的用户名/skill-community` |
| 分支 | `main` |
| 技能目录 | `skills` |
| GitHub Token | 刚才复制的 Token |
| 作者名称 | 你的昵称 |

点击 **💾 保存配置**。

### 5.4 上传技能

社区页面 → **📤 上传/投稿** 标签：

你会看到所有本地技能的列表，每个技能旁边有 **📤 上传** 按钮。

点击上传，技能就会推送到你的 GitHub 仓库中！

### 5.5 告诉别人你的仓库地址

把你的仓库地址（如 `你的用户名/skill-community`）告诉朋友或发到群里。

别人按照 **[第四步](#第四步订阅其他人的技能库)** 添加你的仓库地址就能看到你分享的技能了！

## 进阶：Issue 投稿

如果你想给**别人的**仓库投稿技能，但没有他们仓库的写权限，可以通过 Issue 投稿。

### 操作步骤

1. 社区页面 → **📤 上传/投稿** 标签
2. 滚动到底部「📝 通过 Issue 投稿」区域
3. 填写目标仓库地址（如 `zhangziyana007-sudo/skiller-community`）
4. 选择要投稿的技能
5. 点击 **📝 提交投稿**

这会在目标仓库中创建一个 GitHub Issue，包含你的技能内容。

仓库管理员审核通过后，会把你的技能收录到社区中。

### 查看投稿状态

社区页面 → **📝 投稿列表** 标签，可以看到：

- 🟡 **待审核** — 等待管理员处理
- ✅ **已通过** — 技能已被收录
- ❌ **已拒绝** — 未被采纳

## 常见问题

### Q: 我不想创建仓库，只想用别人的技能
**A:** 完全可以！你不需要配置任何东西。直接在「📡 订阅外部GitHub仓库」中添加别人的仓库地址，就能浏览和安装技能。

### Q: 安装的技能在哪里？
**A:** 安装在 `~/.cursor/skills/技能名/SKILL.md`。Cursor 会自动识别这个目录。

### Q: 技能广场和社区有什么区别？
**A:**
- **🏪 技能广场** — 浏览 Anthropic 官方等知名仓库的技能，类似"应用商店"
- **🌍 社区** — 浏览你订阅的个人/团队仓库，类似"朋友圈"

### Q: 上传技能需要什么？
**A:** 需要一个你拥有写权限的 GitHub 仓库 + GitHub Token（repo 权限）。

### Q: Token 安全吗？
**A:** Token 保存在你本地的 `~/.cursor/skiller/data/community_config.json`，不会上传到任何地方。源代码中也不包含任何 Token。

### Q: 怎么删除已安装的技能？
**A:** 删除 `~/.cursor/skills/技能名/` 文件夹即可。然后在 Dashboard 点击 🔄 重新扫描。
